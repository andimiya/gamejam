Airship CLI

Right now, Airship CLI tools are available for Mac & Linux operating systems. We will be releasing Windows CLI tools in the near future. To get started developing locally, you will need the CLI (command line interface) tools installed on your machine.

Method 1: Auto Install

In your Terminal, enter this command and follow any prompts that arise.
/bin/sh -c "$(curl -fsS https://install.airshipcms.io)"

Method 2: Manual Install

For full installation instructions you can review the install script at https://install.airshipcms.io and proceed with one of the alternative install methods.

Landing & Serving Your Project

  • Create a local directory for your project.
  • Run:
    airship login [AIRSHIP SUBDOMAIN]
  • Your airship subdomain is lowercase, no spaces. It can be entered as airship login projectname or airship login projectname.airshipcms.io. Either will work.
  • You will be prompted with:
    Currently not in an AirshipCMS project
    Would you like to set the current directory /Users/username/Projects/yourdirectory as the AirshipCMS project root for "yourdomain.airshipcms.io"?
    Enter [yes,y] or [no,n]
  • Answer "y" to proceed.
  • Enter your login credentials when prompted.
  • Run:
    airship land
  • You will be asked if you want to download compartments to your project. Answer "y" to proceed. This will download your base project files to your local development directory.
  • Run:
    airship serve
  • This starts the server that will render all of your project files locally.
  • Open your browser and navigate to http://localhost:9001. The local page should look exactly like this page.

Launching Your Project

  • When you are ready to launch (deploy) your project, run the command:
    airship launch
  • Follow the prompts and fix your files if any errors arise.
